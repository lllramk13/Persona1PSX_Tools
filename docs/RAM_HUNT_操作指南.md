# 蹲点找空闲 RAM · 操作指南（给动手的你）

> 目标：找出 PS1 内存里一块**所有游戏模式都从没用过**的连续 128KB，
> 作为 4096 字库的新家。背景见 README §1「字库容量」、接力手册 §10。
>
> 你只需在 DuckStation 里玩、每个场景存一个档；内存对比全部交给
> `re_tools/ramhunt.py` 自动做。**你不用会看内存、不用懂十六进制。**

---

## 为什么要你亲自玩

我试过用反汇编静态找空闲 RAM——失败了：游戏很多数据是"现场算地址"存的，
代码里看不见，扫出来的"空地"里甚至包括字库自己。**唯一可靠的办法是把游戏
各个场景都跑一遍，看哪片内存全程没被碰过。** 这一步必须真人玩，我替代不了。

原理：DuckStation 的存档文件（.sav）里藏着当时整个 2MB 内存的快照。你在不同
场景各存一个档 = 各拍一张内存照片。脚本把照片叠起来比对：**哪些字节在每张照片
里都是 0 = 从没被初始化 = 空闲。**

## 第 1 步：准备

1. 用 **原版日文盘**（不用测试盘）：
   `Game/P1_PSX/Megami Ibunroku - Persona - Be Your True Mind (Japan) (Rev 1).bin`
2. DuckStation 里加载它，能进游戏即可。
3. 建议先在 设置 里把**存档槽**调多一点，或者用"存到文件"。

---

## 第 2 步：逐场景存档（关键，越全越准）

**存档热键**：DuckStation 默认 `F1`=快速存档、`F2`=切换存档槽；或菜单
`System → Save State → 选一个 slot`。**每个场景换一个不同的 slot 存**，
这样会生成多个 .sav 文件。

要覆盖的场景（尽量都跑到，每个存一次）：

| # | 场景 | 怎么到 |
|---|---|---|
| 1 | 开场对话 | 新游戏，进到有对话框的地方 |
| 2 | 世界地图/移动 | 能在城镇/地图上走动时 |
| 3 | 迷宫内 | 进入任意迷宫，第一人称走廊 |
| 4 | **战斗** | 迷宫里遇敌，进战斗画面（战斗用另一套 overlay，很重要）|
| 5 | 菜单/道具 | 打开状态/道具/装备菜单 |
| 6 | 恶魔交涉 | 战斗里发起交涉对话 |
| 7 | 赌场 Casino | 到 Casino（金币系统在高位 RAM，值得单独测）|
| 8 | 姓名输入 | 新游戏最开始的取名画面（如果还能回去）|

> 越多模式越可靠——某块内存只要**有一个模式用到**就不能给字库。漏测模式 =
> 可能误判成空闲，所以宁可多存几个。存 5-8 个就相当够了。

**存档前先确保画面上有文字**（对话框、菜单字都行）——这样字库已加载进内存，
脚本才能靠它定位。

> `场景1_对话.sav`、`场景4_战斗.sav`。

## 第 4 步：跑脚本（我写好了，一条命令）

```bash
cd /home/mark/Code/RomHacking/P1_Tools
python3 re_tools/ramhunt.py /mnt/c/Users/31885/AppData/Local/DuckStation/savestates/SLPS-00500_*.sav
```
（如果你复制改名了，就把那些文件路径列上去。）

脚本会：
1. 每个存档解压出 2MB 内存、靠 FONT.BIN 定位；打印 `✅ 解出 2MB 主内存`；
2. 逐字节比对，找出所有存档里都是 0 的连续块；
3. 打印**最大的几块全 0 区间**，并告诉你有没有 ≥128KB 的候选。

**你要看的就是最后的「结论」几行**：
- `✅ 找到 ≥128KB 全 0 块：0x80XXXXXX 起` → 把这个地址发给我（或下次会话贴出来），
  我就能把字库搬过去、做 4096 验证盘；
- `⚠ 没有 ≥128KB 全 0 块` → 把上面那张「全 0 连续块」表整个发给我，我们改用
  方案 B（只搬 64KB 的新中文页）或想别的办法。

## 第 5 步（等我做）：哨兵确认

你找到候选地址后，我会做一次"哨兵实验"：往那片内存写个显眼图案、跑一遍全流程，
回来看有没有被游戏覆盖。没被覆盖 = 地址真安全，才正式搬字库。这步防止你漏测的
某个模式偷偷用了那块内存。

## 常见问题

- **脚本报"找不到 FONT.BIN 锚点"**：那个存档存的时候画面没有文字（字库没加载）。
  重新进到有对话/菜单的画面再存。
- **只有 1 个存档**：脚本至少要 2 个才能比对；多存几个。
- **`zstandard` 报错/缺库**：`pip install zstandard`（或用项目的 venv）。
- **不确定存档路径**：`ls /mnt/c/Users/31885/AppData/Local/DuckStation/savestates/`
  看看有哪些 .sav。
