# 方案②·动态字库（按场景换字）设计 + 你的 MIPS 第一课

> 背景：全游戏需 ~3111 字，2048 装不下；RAM 无空闲（7 场景转储实测最大空块仅 23KB）、
> BIOS 只读用不了。唯一正解 = 保持 2048 物理槽，**每场景把要用的字换进去**。
> 关键前提已确认：每场景用字最多 434 个（E0 实测），2048 绰绰有余。

## 总设计：让字库"搭 E 段的顺风车"

复用我们已破解的两样东西——**任意长度回插**（`ebin_rebuild`）+ **脚本 VM**：

```
核心区(槽 0..CORE-1)     开机加载, 永不变。装 假名+高频字+人名用字。
                        ← 全游戏 95%+ 的字在这, 编码到哪都一样
换页区(槽 CORE..2047)    每进一个场景, 刷成本场景的生僻字
                        ← 只有零星生僻字用这里

每个 E 段里打包本场景的生僻字形; 段脚本开头插一条自定义 opcode FF5B,
进场景时 FF5B 把那包字形 memcpy 到换页区 → 再正常显示文本。
```

**妙处**：字形数据坐在 E 段里（`ebin_rebuild` 已能任意加长 + 自动修指针），
不用往光盘加文件、不用读盘。新增的引擎改动只有一个 ~12 行的 memcpy handler。

**存档兼容**：玩家姓名用核心区的字 → 所有场景稳定显示，老存档不乱。

# C 步 · 你的 MIPS 第一课：写 FF5B handler

## 这个 handler 要干的事（一句话）

进场景时，把 E 段里打包的那一小包中文字形，从它在内存里的位置，
**拷贝到字库换页区**。就是一次"内存拷贝"。

## 预习：MIPS 你只需要知道这些

MIPS 是 PS1 CPU 的指令集。它有 32 个"寄存器"(32位的小盒子存数)。这个引擎里：

| 寄存器 | 别名 | 这个 handler 里的用途 |
|---|---|---|
| `$s5` | — | **引擎给的**：指向当前脚本记录(我们的 `FF 5B ..`) |
| `$a0`-`$a3` | 参数 | 我们随便用来存 源地址/目标/计数 |
| `$v0`-`$v1` | 临时 | 搬运字节的中转 |

你只需要这几条指令：

| 指令 | 意思 | 例子 |
|---|---|---|
| `lw  $a0, 4($s5)` | **读**：把 `$s5+4` 处的 4 字节读进 `$a0` | 取源地址 |
| `sw  $v0, 0($a1)` | **写**：把 `$v0` 的 4 字节写到 `$a1+0` | 写目标 |
| `lui $a1, 0x801E` | 把 `0x801E0000` 装进 `$a1`(装高 16 位) | 设目标基址 |
| `ori $a1, $a1, 0xNN` | 把低 16 位或进去(**无符号**) | 补目标偏移 |
| `addiu $a0, $a0, 4` | `$a0 = $a0 + 4`(加立即数) | 指针前进 |
| `bnez $a2, loop` | 若 `$a2 != 0` 跳到 loop | 循环判断 |
| `j 0x800AD218` | 无条件跳到某地址 | 干完交还引擎 |
| `nop` | 啥也不做(填延迟槽,见下) | — |

**一个 MIPS 大坑——延迟槽**：`b`/`j` 跳转指令的**下一条**指令**照样会执行**
(硬件流水线特性)。所以跳转后面常跟 `nop` 或一条"顺手做的"指令。写循环时要留意。

## 已知的固定值（写 handler 直接用）

- 源地址：`lw $a?, 4($s5)` —— FF5B 记录格式是 `FF 5B 00 00 <源指针u32>`，
  源指针在 `$s5+4`（这个指针指向 E 段里那包字形，`ebin_rebuild` 会自动填/修）；
- 目标：`0x801E0000 + CORE*32`（换页区起点，CORE 是核心区字数，A 步定，先记作常量）；
- 计数：`换页字数 * 32` 字节（也是常量，先记作 `SWAP_BYTES`）；
- 干完跳 `0x800AD218`（引擎默认收尾：它会按长度表把 `$s5` 前进、处理跳过标志）；
- 我们会把长度表里 FF5B 改成 8（记录 8 字节），这样收尾能正确前进。

## 你的任务：把伪代码翻成 MIPS

伪代码：
```
源  = 读4字节(s5 + 4)              # E 段里字形包的地址
目标 = 0x801E0000 + CORE*32        # 换页区
剩余 = SWAP_BYTES                  # 要拷的字节数
循环:
    tmp = 读4字节(源)
    源  += 4
    写4字节(目标) = tmp
    目标 += 4
    剩余 -= 4
    若 剩余 != 0 继续循环
跳 0x800AD218                       # 交还引擎
```

试着写成 MIPS（填空练习，写不出来的行留着，我补 + 讲）：
```asm
        lw    $a0, 4($s5)          ; a0 = 源地址
        lui   $a1, 0x801E          ; a1 = 目标高16位
        ori   $a1, $a1, ____       ; a1 += CORE*32 的低16位  ← 为什么用 ori 不用 addiu?
        ori   $a2, $zero, ____     ; a2 = SWAP_BYTES (计数)
loop:
        lw    $v0, 0($a0)          ; v0 = *源
        addiu $a0, $a0, 4          ; 源 += 4
        addiu $a2, $a2, -4         ; 剩余 -= 4
        sw    $v0, 0($a1)          ; *目标 = v0
        bnez  $a2, loop            ; 剩余!=0 → 回 loop
        addiu $a1, $a1, 4          ; ← 这行在延迟槽, 想想为什么放这
        j     0x800AD218           ; 交还引擎
        nop                        ; 延迟槽
```

**思考题**（写 handler 时自己想想，答案问我）：
1. 为什么 `ori $a1,$a1,0xED80` 不能写成 `addiu`？（提示：0xED80 当有符号数是负的）
2. `addiu $a1,$a1,4` 为什么放在 `bnez` 后面（延迟槽）而不是前面？
3. 如果 SWAP_BYTES 不是 4 的倍数会怎样？（我们保证它是，但想想为什么要保证）

## 落地与验证（我做）

1. 你写好 asm → 我用汇编器转成机器码字节，填进注入地址 `0x800BADEC`；
2. 把分发表项 `0x800655E4`（现指默认 handler `0x800AD218`）改成指向 `0x800BADEC`；
3. 长度表 file+`0x5694B`（FF5B 项）从 4 改成 8；
4. **哨兵测试**：做一个 E 段，打包一包"显眼的中文字形"，插 FF5B，实机进那个场景，
   看换页区有没有被正确刷成中文（渲染出来）。对了就说明 handler 通了。

## 你会学到

MIPS 寄存器/load-store/循环/延迟槽、跳转表 hook（改一个指针换掉一条指令的行为）、
代码注入到空白区。这是逆向/ROM 破解最核心的手艺，学会这个 handler，
以后改游戏逻辑就有底了。
